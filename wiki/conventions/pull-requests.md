# Pull Request Conventions

## Title
- Concise, imperative mood (e.g., "Add rate limiting to API endpoints")
- Under 72 characters

## Description Structure

### Summary
1-2 sentence overview of what this PR does and why.

### Changes
- Bullet list of meaningful changes grouped by area
- Don't list every trivial file change â€” group logically

### Testing
How this was tested or should be verified.

### Notes (if applicable)
Breaking changes, migration steps, or reviewer callouts.

## Rules
- Use present tense ("Add", "Fix", "Update")
- Focus on intent, not implementation details in the summary
